### Задача. Добавить справочник типов документов в исполняемый скрипт

#### ОПИСАНИЕ
Добавьте описание здесь.
Добавить в меню типы доукментов 

Проблема;
При создании мастер-документа по-умолчанию указывается project хотя на этом уровне может создавааться и line:
kirillkravcov@MacBook-Pro-Kirill uch-docs % ./uch-create-unified-fixed.sh
=== UCH CREATE UNIFIED FIXED: ЕДИНЫЙ СКРИПТ С ПРАВИЛЬНЫМ YAML ===

=== РЕЖИМ СОЗДАНИЯ ДОКУМЕНТА ===
1 - Мастер-документ (уровень 1, корневой)
2 - Дочерний документ (уровень 2+)
3 - Документ с ручным указанием ID
Ваш выбор (1-3): 1

Введите название проекта:
Тестовый проект
Введите теги через запятую (можно оставить пустым):


=== СОЗДАНИЕ ДОКУМЕНТА ===
ID: 05
Уровень: 1
Название: Тестовый проект
Тип: project
Родитель:  ()
Файл: 05 - Тестовый проект.md

Создать документ? (y/n)

Функциональные требования

- поддержать все существующие типы документов для каждого уровня а так же неиерархические документы (00-04-01-03 типы документов)
- добавить в скрипте  к уровню 3 тип документа snapshot - снимок системы (дополнение к 00-04-01-03 типы документов). 
- добавить в скрипте к уровню 4 тип документа subtask - подзадачи (дополнение к 00-04-01-03 типы документов)
- определив уровень будущего документа в предыдущих шагах(шаге) опросника необходимо предоставлять выбор из соответсвующих типов уровня:
уровень 1
project - 1 (для выбора нужно указать цифру)
line - 2

Уровень 2
component - 1
module - 2
...

Без уровня:
Неиерархические документы (нужен отдельный пункт опросника в "режим создания документа")
Далее тот же пункт с указанием имени
Далее выбор из трех типов документа
idea - 1
reference/source - 2
meeting - 3
чтобы поддержать единый формат ответов на опросник, выбор типа должен быть по указанию цифры, а не имени типа.

- нужно создать отдельные шаблон(ы) для этих типов документов (исходить из описания 00-04-01-03)
- нужно чтобы скрипт выполнял автоматическую генерацию ID для таких типов документа (исходить из описания 00-04-01-03)
