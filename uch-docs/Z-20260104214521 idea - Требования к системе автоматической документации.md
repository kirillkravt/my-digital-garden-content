## ОПИСАНИЕ ИДЕИ

Система документации для меня (как для инди-разработчика) это:

- **Виртуальная команда** (все мысли структурированы)
- **Память проекта** (не забыть, что и зачем делал)
- **План развития** (четкий roadmap)
  
### Цель:

Доработать систему с целью повышения уровня автоматизации рутинных задач по ведению документации до 80% относительно ручных операций.

### Существующие проблемы системы:
- hex-нумерация:
без понимания родителя сложно ориентироваться в числовом коде
нет связи документации и кода
- Нет возможности обновить/изменить ссылки внутри документа. Ссылки при переименовании других документов остаются прежними в смежных документах, что приводит к некорректным связям в визуальном графе.
- Нет расширенной типологии документов. 
- Нет валидации содержимого документов для определения типа.
- Нет возможности исправлять автоматически исправлять конфликт id


### Вопросы по системе в целом:
Целевое состояние: Как должна выглядеть ИДЕАЛЬНАЯ система автоматической документации для UCH?

Идеальная система автоматической документации должна:
- Предоставлять точное состояние проекта As-Is
- Обеспечивать постоянный дельта трекинг изменений
а) система может ответить: "Что изменилось за последнюю неделю?"
б) система может сравнивать версии документации
в) система отслеживает прогресс по задачам

Какие изменения отслеживаются автоматически:
-новые файлы/модули в коде
-изменения в зависимостях
-новые задачи
-изменения статусов документов

#### Создание и изменение документов

Что она должна делать автоматически?
- Автоматическое создание документов по внешним шаблонам
- Автоматическая нумерация документов:
-Определение уровня
-Поиск свободного id
-Id в имени документа

- Автоматическое изменение нумерации
- Исправление нумерации:
-уровень выше/ниже (с последующим изменением содержимого и фронтматтер)
-присвоение другого id и переименование документа 
-изменение типа документа

- Ручное создание документа по id/имени
-Решение конфликтов id
-Автоматическое цепочное смещение документов при выборе существующего id
